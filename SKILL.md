# RFP 분석 스킬 (Skill 1)

## 개요
이 스킬은 발주처에서 제공한 **PDF 포맷의 RFP(제안요청서)**를 분석하여 제안서의 전체 구조(**Backbone**)와 핵심 작성 요구사항을 추출합니다. 특히 PDF 텍스트 파싱 기술을 활용하여 계층적 목차(depth 1, depth 2)를 자동으로 생성하는 데 최적화되어 있습니다.

> **용어 정의**
> - **depth (계층/깊이)**: RFP 문서의 목차 계층 구조. 이 스킬에서 추출하는 Lv1/Lv2가 여기에 해당한다.
>   - **Lv1 (depth 1)**: 대분류 섹션 (예: "1. 사업 개요", "2. 기술 요구사항")
>   - **Lv2 (depth 2)**: Lv1 하위의 세부 항목 (예: "1.1 배경 및 목적", "2.3 보안 요구사항")
> - **level (스타일 레벨)**: hwpx-writer가 HWPX 렌더링에 사용하는 폰트·기호 설정 (level 1~6). 이 스킬과 무관하며, proposal-writing 스킬에서 사용한다.

## 주요 기능
1. **Backbone 자동 생성**: PDF 내의 제목 패턴(1., Ⅰ., ①, ■ 등)을 분석하여 제안서의 전체 뼈대(depth 1/Lv1, depth 2/Lv2)를 구성합니다.
2. **계층 구조 정밀도 향상**: 단어 단위로 쪼개진 항목을 병합하거나 재분리하여 논리적인 문단 구조로 정제합니다.
3. **요구사항 파악**: 기술 스펙, 관리 요구사항 등 제안서에서 반드시 답변해야 할 핵심 키워드를 리스트업합니다.

## 사용 방법
- `pdf-parse` 엔진을 사용하여 PDF 내의 텍스트와 메타데이터를 추출합니다.
- 추출된 텍스트에서 휴리스틱 알고리즘을 통해 depth 1/Lv1(주요 섹션)과 depth 2/Lv2(세부 항목)를 구분합니다.
- 분석된 구조는 사용자가 직접 **[위와 합치기]**, **[재분리]**, **[자동 병합]** 기능을 통해 최종 확정합니다.

## 주의 사항
- 텍스트가 아닌 이미지 형식으로 구성된 PDF의 경우 뼈대 추출이 제한될 수 있습니다.
- 추출된 뼈대는 반드시 사용자의 검토를 거쳐 최종 작성본으로 확정하여 워크벤치로 전달해야 합니다.
